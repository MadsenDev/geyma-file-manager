Critical bugs

all 5 fixed — July 5
01

Working Sets break on move & rename

Fixed

moveEntries, commitRename and trashItems never update setDefs references. Rename roadmap.pdf and "Ledger redesign" shows a missing ghost.

Fixed — set references now update on move, rename, trash, restore and permanent delete, and every undo reverses them too.

02

Undo stack only holds one entry

Fixed

pushUndo reads this.state.undo — which is never set — instead of this._undo. The stack resets on every operation; a second Ctrl+Z always says "Nothing to undo."

Fixed — pushUndo now reads this._undo; the stack keeps the last 20 operations.

03

Cut-paste undo loses files

Fixed

The cut-mode undo branch in paste() is dead code — it builds { name: ref.name } and discards it. Undo removes the pasted copies but never restores the originals: the files are gone.

Fixed — undo now restores cut originals (including folder contents) and removes the pasted copies.

04

Status-bar toggle hides the whole bottom zone

Fixed

hasBottom gates on showStatus. With the "Bottom dock" preset — nav, location and search all live in the bottom zone — switching off "Status bar" removes your entire toolbar. The toggle should gate only the status module.

Fixed — the toggle now hides only the status module; other bottom-dock modules stay.

05

Center zone can crush Files to a sliver

Fixed

With several stretch modules stacked in center (e.g. title / files / details / mood), overflow:hidden plus competing flex:1 leaves the file grid showing one clipped row, and Details cuts names mid-glyph. Observed live at ~920×540. Zones need per-module min-heights, or the un-split center should scroll.

Fixed — the center zone now scrolls, and stretch modules keep min-heights (Files ≥ 200px).

Broken at smaller widths

06

Edit-mode bar swallows the screen

Fixed

Below ~950px the hidden-module chips wrap into a tall vertical stack that covers the center of the window. Cap the chip area's height or let it scroll horizontally.

Fixed — chips now scroll horizontally in a single row.

07

Selected tiles clip at zone edges

Fixed

The selection ring and glow get cut off by the zone's hidden overflow — same root cause as 05.

Resolved by the 05 fix — the zone scrolls instead of clipping.

Inconsistencies & dead code

08

Center split doesn't survive reload. Fixed — the split is now restored from the saved layout (split on whenever the lower pane has modules).

09

Vestigial one-button tab bar in the Details panel. Fixed — removed, along with the unused detTab state.

10

"Delete permanently" has no confirmation and no undo. Fixed — now requires the action to be repeated within 4 seconds, with a warning toast.

11

Trashed folders leave orphaned subtrees. Fixed — folder contents now travel to Trash and back on restore; permanent delete removes them.

12

Minor dead code. Fixed — both removed.

Feature gaps

13

No arrow-key navigation. Fixed — arrow keys move the selection in grid (column-aware) and list views, Shift extends the range from the anchor, and the view scrolls to keep the cursor visible.

14

Breadcrumbs aren't drop targets. Fixed — crumbs accept drags with the same highlight treatment as the sidebar.

15

"Open in lower pane" only existed on Places. Fixed — folder items now have it in their context menu too.

16

Escape did two things at once. Fixed — staged: first press closes the menu, second clears the selection.

Visual & accessibility

17

Search placeholder near-invisible on dark skins. Fixed — placeholders now inherit the input's ink color at 45% opacity, correct on every skin.

18

Focus styles removed, nothing added back. Fixed — :focus-visible now draws a 2px currentColor outline on inputs, selects and buttons (mouse clicks stay outline-free).

19

Faint mono meta text ran below comfortable contrast on Parchment and Amber. Fixed — both skins' faint ink darkened (Parchment #A89E88→#7E7359, Amber #77591F→#9A7B36); sizes and dates now read comfortably.

20

Folder mood's raw emoji was louder than the brand voice. Fixed — each mood is now a stroke-drawn glyph in the app's icon style, tinted by the tidiness color, in a soft rounded tile.

What's working well

Coherent skin system

Every surface, module and menu respects the token set across all eight skins plus overrides.

Module drag & drop

Zone hit-testing, drop indicators and the hidden-module palette all behave predictably.

Activity timeline

Move, rename, trash and restore all log correctly and follow files through renames.

List-view container queries

Columns drop out gracefully as the pane narrows — a genuinely nice touch.

Context-aware menus

Items, blanks, trash, places, devices, sets, crumbs and modules each get a tailored menu.

Selection model

Click, Ctrl/Cmd, Shift-range and select-all behave like a native file manager.
